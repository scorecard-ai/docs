- **System-under-test**: this refers to your LLM application that is being tested. Scorecard provides inputs to the system-under-test, which logs responses back to Scorecard.

- **Testset**: a set of Testcases (see below). A Testset usually belongs to a central theme, e.g. "Core", "Customer ABC", or "Adversarial tests".

- **Testcase**: a single tuple of inputs that are passed to the system-under-test. Can also include the ideal or ground-truth response.

- **Ideal Response**: also referred to as ground-truth response. This is provided as the label for the generation, or "what the LLM should produce in the ideal scenario".

- **Must-pass**: a Testset that is intentionally focused on only high-precision Testcases, in order to catch major bugs or regressions. Each deployment or PR should pass this set as an early check, before running more comprehensive or specialized Testsets.

- **Model-graded**: refers to the practice of using LLMs to grade retrieval or LLM outputs.

- **Metric**: can be based on heuristics, model-graded eval, or human eval. A metric has a set of criteria and a single score, either binary or numeric.
