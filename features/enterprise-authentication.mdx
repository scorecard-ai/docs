## Setup Guide

### Configure SAML SSO

<Steps>
  <Step title="Contact Scorecard Support">
    Reach out to enable enterprise authentication for your organization:
    - Email: support@scorecard.io
    - Include: Organization name, desired SSO provider, estimated user count
  </Step>
  
  <Step title="Provide SAML Metadata">
    Share your IdP's SAML metadata with Scorecard:
    ```xml
    <!-- Example metadata structure -->
    <EntityDescriptor entityID="https://your-idp.com">
      <IDPSSODescriptor>
        <SingleSignOnService 
          Binding="urn:oasis:names:tc:SAML:2.0:bindings:HTTP-Redirect"
          Location="https://your-idp.com/sso" />
        <KeyDescriptor use="signing">
          <KeyInfo><X509Data>...</X509Data></KeyInfo>
        </KeyDescriptor>
      </IDPSSODescriptor>
    </EntityDescriptor>
    ```
  </Step>
  
  <Step title="Configure Your IdP">
    Add Scorecard as a service provider in your IdP:
    - **Entity ID**: `https://app.scorecard.io`
    - **ACS URL**: `https://app.scorecard.io/sso/saml/callback`
    - **SLO URL**: `https://app.scorecard.io/sso/saml/logout`
    - **Name ID Format**: Email address
  </Step>
  
  <Step title="Test and Activate">
    Scorecard team will configure and test the integration before activation
  </Step>
</Steps>

### User Experience

Once SAML SSO is configured:

1. **Login Flow**: Users visit app.scorecard.io and click "Sign in with SSO"
2. **Domain Detection**: Enter email domain to redirect to appropriate IdP
3. **IdP Authentication**: Complete authentication with your organization's IdP
4. **Automatic Provisioning**: Account created automatically on first login
5. **Organization Access**: Immediate access to organization's Scorecard projects

## API Access with SSO

### API Key Management

Enterprise users can manage API keys through the dashboard:

<CodeGroup>
```python Python
import scorecard

# Use organization API key for programmatic access
client = scorecard.Client(
    api_key="sk_live_org_abc123...",  # Organization API key
    base_url="https://api.scorecard.io"
)

# All API calls inherit organization context
projects = client.projects.list()
```

```javascript JavaScript
import Scorecard from '@scorecard/client';

// Initialize with organization API key
const client = new Scorecard({
  apiKey: 'sk_live_org_abc123...',  // Organization API key
  baseUrl: 'https://api.scorecard.io'
});

// Organization context automatically applied
const projects = await client.projects.list();
```

```bash cURL
# Use organization API key in requests
curl -X GET https://api.scorecard.io/v2/projects \
  -H "Authorization: Bearer sk_live_org_abc123..." \
  -H "Content-Type: application/json"
```
</CodeGroup>

## Troubleshooting

### Common SSO Issues

**Login Failures**:
- Verify SAML metadata is correctly configured
- Check that user email domain matches configured domain
- Ensure IdP is sending required attributes
- Validate certificate hasn't expired

**Permission Errors**:
- Confirm user is assigned to Scorecard application in IdP
- Check organization membership in Scorecard dashboard
- Verify role mapping from SAML attributes

**Session Problems**:
- Clear browser cookies and try again
- Check if user has multiple active sessions
- Verify session timeout settings
