Claude Code is Anthropic's coding agent, and the Claude Agent SDK is Anthropic's framework for building your own AI agents on top of it. OpenTelemetry tracing is built into Claude Code itself, so the same environment variables trace everything it powers: interactive `claude` sessions, headless `claude -p` runs, and Claude Agent SDK (Python v0.1.18+, TypeScript v0.1.71+) applications, which run Claude Code under the hood.

This quickstart shows how to send those traces to Scorecard for observability, debugging, and evaluation—no code changes required.

<Info>
  **Using the standard Anthropic SDK?** Check out the general [Tracing Quickstart](/intro/tracing-quickstart) for the proxy method that works with any Anthropic client.
</Info>

## Steps

<Steps>
  <Step title="Set up environment variables">
    Configure the OpenTelemetry exporter to send traces to Scorecard. You'll need your Scorecard API key from [Settings](https://app.scorecard.io/settings).

    The variables are the same everywhere—only where you set them differs. Pick the scope that fits:

    <Tabs>
      <Tab title="Shell">
        Export in your terminal to trace the current shell session, or add the exports to your shell profile (`~/.zshrc`, `~/.bashrc`) to trace every session on your machine:

        ```bash
        export OTEL_EXPORTER_OTLP_HEADERS="Authorization=Bearer <your_scorecard_api_key>"
        export ENABLE_BETA_TRACING_DETAILED=1
        export BETA_TRACING_ENDPOINT="https://tracing.scorecard.io/otel"
        export OTEL_LOG_USER_PROMPTS=1
        export OTEL_LOG_TOOL_DETAILS=1
        export OTEL_LOG_TOOL_CONTENT=1
        export OTEL_RESOURCE_ATTRIBUTES="scorecard.project_id=<your-project-id>"
        ```
      </Tab>
      <Tab title="Claude Code settings">
        Add an `env` block to `.claude/settings.json` in your repository to trace every Claude Code session in that project, or to `~/.claude/settings.json` to trace all projects:

        ```json .claude/settings.json
        {
          "env": {
            "OTEL_EXPORTER_OTLP_HEADERS": "Authorization=Bearer <your_scorecard_api_key>",
            "ENABLE_BETA_TRACING_DETAILED": "1",
            "BETA_TRACING_ENDPOINT": "https://tracing.scorecard.io/otel",
            "OTEL_LOG_USER_PROMPTS": "1",
            "OTEL_LOG_TOOL_DETAILS": "1",
            "OTEL_LOG_TOOL_CONTENT": "1",
            "OTEL_RESOURCE_ATTRIBUTES": "scorecard.project_id=<your-project-id>"
          }
        }
        ```

        <Warning>
          `.claude/settings.json` is usually committed to version control. Keep your API key out of it—put the `env` block in `.claude/settings.local.json` (Claude Code's personal, not-checked-in settings file) instead.
        </Warning>
      </Tab>
      <Tab title="Agent SDK (in code)">
        Pass the variables per `query()` call via the `env` option—useful when one application traces to different projects or you don't control the process environment:

        <CodeGroup>

        ```typescript agent.ts
        import { query } from "@anthropic-ai/claude-agent-sdk";

        const stream = query({
          prompt: "What is 2 + 2?",
          options: {
            env: {
              ...process.env, // env replaces the subprocess environment; keep existing vars
              OTEL_EXPORTER_OTLP_HEADERS: `Authorization=Bearer ${process.env.SCORECARD_API_KEY}`,
              ENABLE_BETA_TRACING_DETAILED: "1",
              BETA_TRACING_ENDPOINT: "https://tracing.scorecard.io/otel",
              OTEL_LOG_USER_PROMPTS: "1",
              OTEL_LOG_TOOL_DETAILS: "1",
              OTEL_LOG_TOOL_CONTENT: "1",
              OTEL_RESOURCE_ATTRIBUTES: "scorecard.project_id=<your-project-id>",
            },
          },
        });
        ```

        ```python agent.py
        import os
        from claude_agent_sdk import query, ClaudeAgentOptions

        stream = query(
            prompt="What is 2 + 2?",
            options=ClaudeAgentOptions(env={
                **os.environ,  # env replaces the subprocess environment; keep existing vars
                "OTEL_EXPORTER_OTLP_HEADERS": f"Authorization=Bearer {os.environ['SCORECARD_API_KEY']}",
                "ENABLE_BETA_TRACING_DETAILED": "1",
                "BETA_TRACING_ENDPOINT": "https://tracing.scorecard.io/otel",
                "OTEL_LOG_USER_PROMPTS": "1",
                "OTEL_LOG_TOOL_DETAILS": "1",
                "OTEL_LOG_TOOL_CONTENT": "1",
                "OTEL_RESOURCE_ATTRIBUTES": "scorecard.project_id=<your-project-id>",
            }),
        )
        ```

        </CodeGroup>
      </Tab>
    </Tabs>

    <Note>
      Replace `<your_scorecard_api_key>` with your actual Scorecard API key (starts with `ak_`), and `<your-project-id>` with the project you want traces to land in.
    </Note>

    <Warning>
      If the project ID is wrong or invalid, traces will be sent to the **oldest project** in your organization. If the API key is missing or invalid, traces are dropped silently—nothing appears in Scorecard and no error is raised.
    </Warning>
  </Step>

  <Step title="Run Claude Code or your agent">
    With the environment variables configured, everything Claude Code does is traced automatically. Run an interactive session, a headless prompt, or a Claude Agent SDK application:

    <CodeGroup>

    ```bash Claude Code CLI
    # Interactive sessions and headless -p runs are both traced.
    claude -p "What is 2 + 2?"
    ```

    ```python example.py
    import anyio
    from claude_agent_sdk import (
        AssistantMessage,
        TextBlock,
        query,
    )

    async def main():
        async for message in query(prompt="What is 2 + 2?"):
            if isinstance(message, AssistantMessage):
                for block in message.content:
                    if isinstance(block, TextBlock):
                        print(f"Claude: {block.text}")

    anyio.run(main)
    ```

    </CodeGroup>
  </Step>

  <Step title="View traces in Scorecard">
    Navigate to the [**Records**](/features/records) page in [Scorecard](https://app.scorecard.io) to see your agent traces. Click any record to open its trace detail panel, which breaks the trace into four tabs: **Conversation**, **Timeline**, **Chat**, and **Debug**.

    <DarkLightImage lightSrc="/images/records-table-claude-sdk-light.png" darkSrc="/images/records-table-claude-sdk-dark.png" alt="Records page with a record selected, showing the trace detail panel" caption="Selecting a record opens its trace detail panel." />

    The **Conversation** tab shows a chat-like replay of user messages, assistant responses, and tool calls.

    <DarkLightImage lightSrc="/images/claude-sdk-conversation-view-light.png" darkSrc="/images/claude-sdk-conversation-view-dark.png" alt="Conversation tab showing user messages, assistant responses, and tool call results" caption="Conversation tab of an agent trace." />

    The **Timeline** tab shows a Gantt chart-style breakdown of every span, so you can see how LLM calls and tool executions overlap and how long each takes.

    <DarkLightImage lightSrc="/images/claude-sdk-trace-timeline-light.png" darkSrc="/images/claude-sdk-trace-timeline-dark.png" alt="Timeline tab showing span hierarchy with duration bars for LLM calls and tool executions" caption="Timeline tab of an agent trace." />

    The **Chat** tab lets you ask questions about the trace in natural language—what happened, why a step failed, or how it performed—and compare records side by side.

    <DarkLightImage lightSrc="/images/claude-sdk-chat-view-light.png" darkSrc="/images/claude-sdk-chat-view-dark.png" alt="Chat tab answering a natural-language question about the trace" caption="Chat tab: ask questions about a trace." />

    The **Debug** tab pairs the full span tree with a trace overview—duration, cost, tokens, and model—plus the raw span data for deeper inspection.

    <DarkLightImage lightSrc="/images/claude-sdk-debug-view-light.png" darkSrc="/images/claude-sdk-debug-view-dark.png" alt="Debug tab showing the span tree alongside the trace overview dashboard" caption="Debug tab: span tree and trace overview." />
  </Step>
</Steps>
